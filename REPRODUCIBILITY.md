# Reproducibility Guide

## Seeds and Configuration

All experiments use deterministic seeds. Results are reproducible within the same hardware + software environment.

### ExactK Production Config

| Parameter | Value |
|-----------|-------|
| `delta_k` | 0 (exact K matching) |
| `lambda_iso` | 0.10 |
| `margin` | 0.30 |
| `lambda_decay` | 0.85^(ep−8) for ep > 8 |
| `max_pairs` | 256 (d=5) / 512 (d=7) |
| `warmup_epochs` | 5 |
| `total_epochs` | 12 |
| `effective_batch_size` | 256 (micro=64, grad_accum=4 for d=7) |

## Hardware Requirements

### Development (d=5)
- CPU: Any modern multi-core
- RAM: 16 GB minimum
- GPU: NVIDIA GPU with ≥ 4 GB VRAM (optional, CPU works but slower)

### Production (d=7)
- CPU: Multi-core Xeon or equivalent
- RAM: 64 GB recommended (188 GB used in original experiments)
- GPU: NVIDIA GPU with ≥ 16 GB VRAM
- Original hardware: RunPod cloud (Intel Xeon 6952P, NVIDIA RTX PRO 6000 96 GB)

## Software Environment

- Python 3.10+
- PyTorch (CPU or CUDA)
- See `requirements.txt` for full dependency list

## Reproducing Results

### Quick Validation (d=5, 2 seeds, ~10 min)

```bash
bash scripts/reproduce/repro_day69_d5.sh
```

### Full d=5 Reproduction (10 seeds, ~2 hours)

```bash
bash scripts/reproduce/repro_day69_d5.sh --full
```

### d=7 OOD Reproduction (requires GPU, ~4 hours)

```bash
bash scripts/reproduce/repro_day70_d7.sh --full
```

### Holdout Reproduction (requires GPU, ~4 hours)

```bash
bash scripts/reproduce/repro_day75_holdout.sh --full
```

### Post-Training KPI Computation

```bash
bash scripts/reproduce/run_selector_v6.sh
```

## Floating-Point Determinism

Results are deterministic within the same environment. Cross-platform differences may cause minor floating-point divergence (< 1% relative) due to:
- Different CUDA versions
- Different CPU architectures (x86 vs ARM)
- PyTorch version differences

The **qualitative conclusions** (PASS/FAIL, direction of improvement) are robust across platforms.

## Artifact Policy

This repository contains **code and documentation only**. Generated artifacts (model checkpoints, JSONL logs, KPI reports) are excluded by `.gitignore`. All artifacts are regenerated by the reproduction scripts.
